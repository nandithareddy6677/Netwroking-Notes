# 📖 Introduction

Organizations often need employees to securely access internal resources from remote locations.

A Virtual Private Network (VPN) provides a secure, encrypted connection over the public Internet.

---

# 🔒 What is a VPN?

A VPN (Virtual Private Network) creates an encrypted tunnel between a user and a private network.

Benefits:

- Secure Communication
- Data Encryption
- Privacy Protection
- Secure Remote Access

---

# VPN Types

## Remote Access VPN

Allows individual users to securely connect to an organization's network.

Example:

Employees working from home.

---

## Site-to-Site VPN

Connects two or more office networks securely.

Example:

Hyderabad Office ↔ Bengaluru Office

---

# VPN Tunneling

VPN encapsulates data into secure tunnels.

```
User

↓

Encrypted Tunnel

↓

VPN Gateway

↓

Company Network
```

# 🌍 Real-Life Example

An employee working from home connects to the company's VPN before accessing confidential files.

The VPN encrypts all traffic, preventing attackers from reading sensitive information.

---

# 💡 Best Practices

- Use MFA.
- Use modern VPN protocols.
- Rotate VPN credentials.
- Monitor VPN logs.
- Disable unused VPN accounts.

# 📌 Key Takeaways

- VPN creates encrypted tunnels.
- Remote Access VPN connects individual users.
- Site-to-Site VPN connects offices.
- Cloud providers offer managed VPN services.

# ❓ Interview Questions

### 1. What is VPN?

A secure encrypted connection over a public network.

### 2. Difference between Remote Access VPN and Site-to-Site VPN?

Remote Access connects users, while Site-to-Site connects networks.

### 3. Name three VPN protocols.

- IPSec
- OpenVPN
- WireGuard

### 4. Which AWS service provides managed VPN?

AWS Site-to-Site VPN.

### 5. Why is VPN important?

It protects data transmitted over public networks.
