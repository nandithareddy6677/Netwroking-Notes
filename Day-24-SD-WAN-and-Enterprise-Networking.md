# 📖 Introduction

Large organizations have offices across different cities and countries.

Connecting these offices securely and efficiently is challenging.

SD-WAN modernizes Wide Area Networking using centralized software management.

---

# 🌍 What is SD-WAN?

Software Defined Wide Area Network (SD-WAN) is a software-managed WAN technology that securely connects branch offices, cloud services, and data centers.

---

# Traditional WAN

Uses:

- MPLS
- Dedicated Links

Problems:

- Expensive
- Less Flexible

---

# SD-WAN

Uses:

- Broadband Internet
- MPLS
- LTE/5G

Managed centrally.

---

# SD-WAN Features

- Centralized Management
- Intelligent Routing
- Application Awareness
- Built-in Security
- Automatic Failover

---

# Benefits

- Lower Cost
- Better Performance
- High Availability
- Easy Deployment
- Cloud Integration

---

# Enterprise Networking

Enterprise Networks include:

- Headquarters
- Branch Offices
- Cloud Infrastructure
- Remote Workers

---

# SD-WAN in Cloud

Cloud providers support hybrid connectivity.

Examples:

- AWS Transit Gateway
- Azure Virtual WAN
- Google Cloud Network Connectivity Center

---

# Real-World Example

A multinational company connects offices in India, Singapore, and the UK.

SD-WAN automatically selects the fastest and most reliable path for traffic while maintaining secure communication.

# 📌 Key Takeaways

# ❓ Interview Questions

### 1. What is SD-WAN?

A software-managed WAN solution for connecting distributed networks.

### 2. Name two benefits of SD-WAN.

- Centralized Management
- Lower Cost

### 3. Why do organizations use SD-WAN?

To improve performance, simplify management, and securely connect multiple locations.
